通过claude code编写网站教程

2026-03-08 · Tech · 中文

这篇文章记录我用 Claude Code 搭建纯静态博客的过程，目标是：不引入构建工具，直接用 HTML / CSS / JS 生成可部署到 GitHub Pages 的站点，并保持结构清晰、便于后续维护。

1. 明确需求与目录结构

首先把需求说清楚：需要首页、关于页、日志列表页、日志详情页，并准备一个技术文章区。然后让 Claude Code 创建基础目录：

  • /assets：统一样式与脚本
  • /diary：日志列表与日志详情
  • /tech：技术文章列表与详情

结构确定后，后续新增页面就可以沿用统一模板，减少重复劳动。

2. 搭建统一页面骨架

用同一套 header / footer 结构，保证全站视觉一致。导航包含“首页 / 关于我 / 日志 / 技术”，并将导航改为左侧固定栏，便于在内容较长时快速跳转。

3. 先做首页，再做列表页

首页使用“最新日志 + 技术文章”两块区域，便于展示更新状态。日志列表页与技术列表页都采用卡片式布局，信息包含：日期、标题、摘要、链接。

4. 写第一篇日志与第一篇技术文

日志内容从真实经历扩展，强调细节与情绪；技术文则记录过程、步骤与关键文件，让之后复用更方便。每篇文章都有独立页面，便于分享链接。

5. 统一样式与响应式

样式集中在 assets/styles.css 中，重点包括：

  • 卡片布局与阴影，提升层次感
  • 字体栈兼容中文与英文
  • 小屏幕自动切换为顶部导航

6. 持续更新的方式

后续只需要：

  1. 新增一篇 HTML 文章页面
  2. 在对应列表页新增一条卡片
  3. 如需首页展示，补充首页列表

这种方式简单直观，不依赖构建环境，也特别适合新手或轻量级站点。

7. 部署到 GitHub Pages

完成后把目录推到 GitHub 仓库，在 Pages 设置中选择 main 分支的根目录即可发布。

以上就是我用 Claude Code 搭建纯静态站点的流程，后续会继续补充模板与写作规范，保持内容更新。
